# DNS Demo Site

Инструкция для пользователя веб-интерфейса DNS-демонстрации.

## Как пользоваться

1. Открой сайт в браузере.
2. В поле домена введи имя (по умолчанию стоит `gov.ru`).
3. Выбери тип записи: `A`, `AAAA`, `MX` или `TXT`.
4. Нажми кнопку проверки DNS-запроса.
5. Посмотри результат в карточках:
   - локальный резолвер (`127.0.0.1`);
   - публичный резолвер (`8.8.8.8`);
   - определенный источник ответа.

## Что показывает интерфейс

- `Ответ` — найденные значения записи.
- `TTL` — время жизни DNS-ответа.
- `AD flag (DNSSEC)` — признак DNSSEC-валидации.
- `Query time` — время ответа в миллисекундах.
- `Источник` — предполагаемый источник ответа:
  - `local-zone`
  - `Redis`
  - `кэш Unbound`
  - `интернет`
  - `н/д` (если локальный резолвер недоступен)

## Кнопки управления

- `Сбросить кэш Unbound` — очищает кэш Unbound.
- `Сбросить кэш Redis` — очищает кэш Redis.

После нажатия кнопок можно сразу повторно выполнить тот же DNS-запрос и сравнить, как меняются `Query time` и `Источник`.

## Если видишь ошибку

- Если в локальном блоке отображается `Ошибка`, значит локальный резолвер или доступ к нему недоступен.
- Если запрос к `8.8.8.8` успешный, а локальный нет — проблема обычно на стороне локального DNS/прав доступа.
